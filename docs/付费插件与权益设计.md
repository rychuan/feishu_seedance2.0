# 付费插件与权益设计

## Context

当前项目是飞书多维表格字段捷径，用户自己填入火山引擎 API Key 调用 Seedance 2.0 生成视频。要将其变为付费产品，需要：移除用户自带的 API Key（由开发者统一管控）、接入飞书支付通道、按权益控制使用。

代码中已有权益检查雏形（`src/index.ts:205-209`）：
```typescript
const isNeedPayPack = context?.isNeedPayPack === true;
const hasQuota = context?.hasQuota === true;
if (isNeedPayPack && !hasQuota) {
  return { code: FieldCode.Success as const, data: '⚠️ 请先购买会员包后再使用' };
}
```

---

## 一、收费模式选择

飞书支持 4 种付费方案（选一种，不能混用）：

| 方案 | 适合场景 | 定价示例 |
|------|---------|---------|
| **按人付费** | 团队/企业，每新增一人付费 | 基础版 29 元/人/月，专业版 59 元/人/月 |
| **按企业付费** | 固定人数套餐 | 基础版 290 元/月（10 人），专业版 590 元/月（10 人） |
| **增值包** | 按次计费，适合低频使用 | 50 次/49 元，200 次/159 元 |
| **一次性收费** | 买断 | 999 元永久 |

**推荐：增值包模式** — 视频生成按次消费，用户理解成本低，和 Seedance API 的按次计费对齐。

## 二、权益分级设计

| 等级 | 月生成次数 | 最高分辨率 | 视频时长 | 多模态输入 | 价格 |
|------|-----------|-----------|---------|-----------|------|
| 免费试用 | 3 次/月 | 480p | ≤5s | 仅文本 | 0 元 |
| 基础版 | 50 次/月 | 720p | ≤8s | 文本+图片 | 49 元/月 |
| 专业版 | 200 次/月 | 1080p | ≤15s | 全模态 | 129 元/月 |
| 旗舰版 | 不限 | 2K | ≤15s | 全模态+优先队列 | 299 元/月 |

## 三、技术架构

### 方案 A：纯飞书支付（推荐起步方案）

```
用户触发生成 → execute()
  ├── context.isNeedPayPack / context.hasQuota → 权益校验
  ├── 本地次数计数（存在多维表格隐藏列或环境变量）
  ├── 用开发者统一的 API Key 调用 Seedance
  └── 返回结果
```

**改动点**：
1. 移除表单中的 `apiKey` 字段，改为环境变量/配置文件存储
2. 在 `execute()` 中根据付费方案限制分辨率、时长等参数
3. 使用 `context` 中的权益信息做功能门控

**优点**：改动最小，飞书负责收款
**缺点**：无法精确计次（飞书不提供使用量回调），权益粒度有限

### 方案 B：后端代理（完整控制方案）

```
用户触发生成 → execute()
  ├── context.fetch() → 你的后端 /api/generate
  │     ├── 后端校验权益（查数据库）
  │     ├── 后端调 Seedance API
  │     ├── 后端记录使用次数
  │     └── 返回结果
  └── 返回给用户
```

**改动点**：
1. 新建后端服务（Node.js/Python），暴露 `/api/generate` 和 `/api/query` 接口
2. 后端管理：用户表、权益表、使用记录表、API Key 池
3. 字段捷径简化为：下载附件 → 转发到后端 → 返回结果
4. 白名单加上后端域名

**优点**：精确计次、灵活定价、API Key 安全、可扩展
**缺点**：需要自建服务器、多一跳延迟

## 五、飞书开放平台操作步骤

1. **登录** [飞书开放平台](https://open.feishu.cn/)
2. **创建应用** → 企业自建应用 / 三方应用
3. **开通付费能力** → 应用能力 → 付费方案
4. **配置方案**：选择"增值包"模式，设置方案名称、价格、使用次数，设置 7 天试用期
5. **配置环境变量** → 将 Seedance API Key 存为应用环境变量
6. **提交审核** → 上架到字段捷径中心

## 六、验证方式

1. 在飞书开放平台配置测试付费方案
2. 用测试企业安装字段捷径
3. 验证 `context.isNeedPayPack` / `context.hasQuota` 返回值
4. 验证未购买用户看到付费引导提示
5. 验证已购买用户正常生成视频
6. 验证参数降级逻辑（如免费用户选 1080p 自动降级为 480p）

---

## 七、建议路径

| 阶段 | 方案 | 时间 | 目标 |
|------|------|------|------|
| MVP 验证 | 方案 A（纯飞书支付） | 1-2 天 | 验证用户愿不愿意付费 |
| 增长期 | 方案 B（加后端） | 1-2 周 | 精确计次、灵活定价 |
| 规模化 | 方案 B + API Key 池 + 队列 | 持续 | 降低成本、提升体验 |

先做方案 A 验证可行性，用户量上来后再迁移到方案 B。
